tl;dr - scp is a pretty cool command for frequent ssh-ers.

For those of you who have to ssh into your school directories, this is a pretty useful trick. Being a student at Rutgers, I find myself constantly ssh-ing into the ilab machines. So today moring, I had to finish up an assignment I started last night and submit it before 6:40pm today. So I ssh’ed, tmux/vim’d, and finished. Then I discovered a problem: how am I going to submit this? Those of you unfamiliar with Rutgers’ Sakai submissions, we have to submit our files through a site called Sakai. There is no "which server would you like to upload from" kind of thing built into it (though, that would be really nice), so it can only upload from your own directory. I had my file on an ilab server.

Thanks to Google, found a command called scp. What scp does is take a file from an ssh-able server and spits it out in any local directory you want. After trying to figure out what the man page was saying, I found thisÂ which is basically the man page for dummies.

scp USERNAME@SERVER:PATH_NAME_OF_FILE LOCAL_DIRECTORY

Should, after entering in whatever credentials needed for that server/login, drop the file you want in your LOCAL_DIRECTORY. Super useful, now I can submit my assignment, woo.
