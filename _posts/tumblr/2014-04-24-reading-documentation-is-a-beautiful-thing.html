So, a week or two ago at work, I realized there was something wrong with my Zend app that is supposed to access the Google Calendar API (GData, as it is built into Zend). Whenever I had recurring events, I would only get the first instance of that event in my range of search. I told my boss, he told me to fix it (naturally), and so for a week and a half I’ve been developing some way to handle this. I’ve been parsing the recurrence RRULE tag in my associative array I get back from Google, checking each day to see if it’s a recurrence date, storing it all into massive arrays, etc. 

Then today, I was reading through the GData Calendar EventQuery documentation (like the actual php file because I wanted the descriptions of each function. Note that the names are all listed here, this buried Zend-treasure) and I found something called getSingleEvents(). Now this got me thinking something along the lines of “oh shit, is this what I think it is, is this the recurrence handler” because I actually did hear about this somewhere in the Zend API comments, but chose to ignore it. 

Anyways, this ended up not working, but then after some googling, I found that setParams(‘singleevents’,’true’) worked and I was like OWWEwepfoakjweopkaweroiEIWEFjpOWEJIPWEkf to say the least.

So, I told my boss, kind of sad, really sad actually. Scrapped my poor code to handle recurring events, I’m probably more sad because of that.

Lesson learned? always look through documentation before doing something retarded like implementing your own recurrence handler. smh.
