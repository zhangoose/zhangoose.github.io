Well, if by life I mean productivity, yes. 

So a little background information: I usually code using a combination of tmux and vim, to say the least. I work in the Rutgers SAS-IT office on College Ave, whose office is actually just a really small house. Over there, I make super small webapps to do super easy stuff like parse through log files and (what I’m working on right now), calculating the time between two dates accounting for workhours and holidays. To do that, I have to SSH into a server. However, that server is super basic and doesn’t have anything (and can’t have anything) I need to work (tmux and an editable vimrc). Well shit, there goes my productivity right?

I know that server file system and I guess the basic ZendFramework setup (well, at least the jail cell I’m limited to) super well now. Why? I kept cd ../../../………-ing EVERYWHERE all the time to get places, then I vim some file, then I forget what I was trying to add to that file, so I have to cd away from that file to vim the previous file I was at, only to cd back to the directory whose file I was going to edit. That was basically how I got things “done” for a good 2 months (oh god, the shame). 

But then the other day, my coworker was observing me cd-ing about and mentioned that I should try to mount the server to a local directory. He explained that if you mount the server to a local directory, you can edit your server files from that local directory and it will be changed on the server, so less SSH-ing. 

That. was. fucking. awesome. No, seriously, I was like “what the fuck, you can do that?!?!?!” So I proceeded to do that and OHMYGAWD I HAVE TMUX. I HAVE A COOL VIMRC. ]

So before this turns into a completely fruitless post, here is the tutorial I used to mount my server. This is actually the ubuntu way (since I’m running elementaryOS on my work computer), so it’s probably different for you guys. 

But yeah, pretty awesome shit, definitely check it out if you’re as unfortunate as I to not know such a thing existed.
