So I was sitting in my Internet Technology class one day, super bored, decided that my handwriting deteriorated a little too much during winter break, and looked to my netbook for taking notes instead. I usually use vim and just do a vim 0129 where the filename would just be whatever date it was that day. And that totally worked most of the time unless there were super complex diagrams involved.

But then I found this cool thing called Markdown (which I actually knew before through Github, but didn’t really pay attention to it until recently). More specifically, Taking Notes With Markdown. ….. And then after that, I discovered the art of Making Blog Posts With A Markdown To HTML Translator.

I’ll start with why it’s really nice to take notes with Markdown. A little background: I never use Windows unless it’s on a school computer (or for work); I am, for many reasons, a Linux user. Therefore, I never really have Microsoft Office and its user-friendly-ish What-You-See-Is-What-You-Get editor around. Sure, I have Libre Office around, but trying to learn how to use 2 rich text editors that are pretty much the same in functionality but different in where certain properties are is actually really annoying.

And there’s Markdown. I knew some of my friends in Computer Science take notes with Markdown (one of them does LaTeX, which is pretty awesome), but I never really got into it because I was like "oh god, I don’t really feel like learning this syntax" (which is actually really really simple). So anyway, long story short, yes, use Markdown to take notes if most of what you write is going to be straight-up English (and content). And maybe not a lot of diagrams. And here’s why:

Why you should use Markdown to take notes?

  • Syntax is mad easy.
  • Way faster than dropping the drop-down bar for the headers.
  • I do what I want, red lines. If I say malloc is a word, then malloc is a word.
    • If I want to make the first letter in that line lowercased, no auto-capitalizing it.
  • More organized than plain .txt files (for you people who take notes in vim 0129.
  • HTML/PDF convertable.
  • Esp. for Computer Science students: you can make tags.
    • Teacher puts code block on board, you type it out in your non-markdown editor, it blends in with the rest of your text, if only it looked like actual code like:

char * p = (char *)malloc(sizeof(char)*10));

So now that you know why I think you should use Markdown to take notes, I’m going to move on to the main point I’m trying to make here.

Why you should use Markdown to make blog posts?

  • HTML tags are really annoying to type out.
  • More options, sometimes.

Here’s a really good cheat-sheet for the Markdown syntax. Believe me, that’s all you need to learn it (and either an editor with live previewing, which is always nice, or if you’re more of a vim/emacs/nano-for-everything user, an interpretter/converter for Markdown).

A couple notes: tables are apparently not supported in original Markdown syntax. There are a couple plug-ins you can use such as Multimarkdown (which, if I remember correctly, also incorporates MathJax).

For iOS users: You guys have a lot of options for text editors. Most of my frieds use Mou. As you can see from the website, it comes with live previewing. But I’ve never used it.

For Linux users: there’s ReText which is free and has live previewing and conversions to HTML/PDF/ODT. Only bad part is that it doesn’t support Multimarkdown (and I don’t think it can, but I’ll update this if I find a way). It’s pretty straight-forward and to-the-point. I would definitely recommend this for someone just starting or, well, anyone who does not feel a need for Multimarkdown. I’m actually using it right now to type up this blog post, and it’s perfect.
