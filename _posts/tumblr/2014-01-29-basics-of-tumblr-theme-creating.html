For Minh

These are just going to be the basics of making a Tumblr theme for yourself. As I said in a previous blog entry, this is a very good resource. But in this entry, I will summarize the basics of that resource to get you started on playing around with your Tumblr theme.

So with any website layout, you need a relatively okay understanding of CSS (or else you end up with my super basic theme).

(btw, trying to make a Tumblr theme is easier if you worked with other types of themes)

A few things you should know about the Tumblr theme you’re going to create:

  • Perhaps one of the very few things that will set your theme apart from everyone else’s is your CSS (of course).
  • Your “HTML” will be for the most part very similar to everyone else’s.
  • Tumblr renders your HTML to figure out how your posts will look.
  • You can style your Text posts, Image posts, Audio posts, Video posts, etc… individually (an advantage of Tumblr’s rendering process).
  • You can let the user of your theme choose some of the properties of your Tumblr theme (ex: colors, font families, sizes, etc).

So, the reason why all of the above are true is because of how Tumblr renders your HTML. Tumblr, in short, will parse through your HTML, look for blocks and variables, and sort all your posts based on the surrounding HTML.

Special Tumblr Syntax

  • block - {block:BlockName}
  • variable - {VariableName}

Here is a resource of the basic BlockNames and VariableNames available. As you can see from this resource, the VariableNames and BlockNames are separated by each type of Post. This is attributed to, as I said earlier, how Tumblr lets you style each of your posts separately.

You can let the user of your theme choose some of the properties such as colors

Tumblr uses things called<meta> tags to 1. tell Tumblr that you want these properties to be customizable and 2. set default values to these properties.

To put it simply, these <meta> tags have 2 attributes: name and content.

  • name is the name you can assign to that particular property somewhere in your HTML.
  • content is the default value you assign to that property. Ex: if name is referring to a colorProperty, you may want to set the content to "#666666".

Alright, let’s actually get into some code.

Again, this is just a basic tutorial, the bare bones of an adequate Tumblr theme. I will leave the CSS to you and only cover the HTML part (because the CSS is a little self-explanatory).

Something you could do right now is to take your favorite Tumblr theme, apply it to your site, go into Customize, Edit HTML, and strip away the CSS. Look over the HTML leftover a bit, remember the syntax for blocks and variables, and you should be able to figure it out from there (or just use that as a model and write your own barebones just with different class/id names, then plop on your styllings).

(Usually the theme coder will organize the code in a way that all the CSS stylings will be located in <head><style> Here </style></head>).

Scroll down a bit to where it says “here’s an example of the complete markup of a theme”, if you don’t want to use the strip-away-stylings method. These are the “barebones” of a Tumblr theme. If you just want a simple normal Tumblr theme, you could just use this as the HTML.

However, if you maybe, I don’t know, don’t want the {Caption}s to show on your Image Posts or don’t want the {Title}s to show on your Text Posts, you may want to edit your barebones a bit so that that variable doesn’t show.

This requires an understanding of how Tumblr renders your theme. Let’s take the most basic Post type there is: Text. Taken from the above resource:


{block:Text}
    <li class="post text">
        {block:Title}
            <h3><a href="{Permalink}">{Title}</a></h3>
        {/block:Title}{Body}
    </li>
{/block:Text}

This block will determine what all of your Text posts will look like.

These, taken from here are the different variables and blocks that apply to Text posts.

Starting from the first line, {block:Text} is like an HTML tag, and it says that anything within the {block:Text} and {/block:Text} tags are going to be the contents of each Text post you have.

So, if I decided to write

 {block:Text} <p>Hi</p> {/block:Text}

If I have 4 text posts, all 4 text posts would look like <p>Hi</p>. Note that in this modification, your actual Text post’s content is not displayed. This is because we didn’t reference the content’s variable name.

On the third line, we see {block:Title}. This block acts very similarly to the {block:Text}. The variable {Title} calls whatever Title your Text post has. It is also wrapped with <a> and <h3> tags. That way, every Text post’s Title will show up wrapped in said tags. The same can be said for {Body}, which displays your Text post’s body.

Going back to how before, we couldn’t see the contents of posts in our Text block, calling the {Title} and {Body} variables tells Tumblr to grab the content in said variables and display it. It’s kind of like having variables but in HTML. It’s also similar to <?php echo($Title); ?> where $Title is a variable corresponding to the Title of each post.

Now, suppose we want the theme user to decide the color of the {Title}, which is wrapped with <h3> tags. You should make a <style></style> block in the header of your HTML.

Generally, if you want to change the color of a <h3>, you would just do:

<style> h3 { font-color: red; } </style>

However, you want the user to be able to edit the font-color field. Do this by using Tumblr’s <meta> tags and variables. There is a very good example of such here, screenshotted below.

There are different kinds of “custom variables” you can create in Tumblr; they are mostly for the custom font/color/if selections. To enable a “custom variable” to be used and changed by the user, add this into your header:

<meta name="type:NAMEOFVAR" content="DEFAULTVALUE"/>

Where type can be replaced with font, color, if.

Going back to our problem of wanting the user to be able to select the color of the <h3> text, the <meta> tag would look like:

<meta name="color:Header3" content="red">

Here, the default color of your <h3> will be red. The CSS should therefore be changed to:

<style> h3 { color: {color:Header3}; } </style>

Otherwise, your changes to the color won’t be applied to the actual <h3>.

Meta names font and color are self-explanatory as to what they refer to. if is used as booleans (true/false values). For example, you can make a boolean variable to control whether or not you want your blog to show tags on your posts. In this case, your content will be either 0 or 1 (0 = false, 1 = true). You would use these variables in the HTML instead of the CSS <style> tags.

Suppose your <meta> tag was the code snippet below and you wanted control on whether you show a cheesey <p>Hi, welcome to my blog</p> message on your page.

<meta name="if:Show Greeting" content="1" />

In your HTML body, this custom boolean variable will be interpretted as a block. You should have:

{block:IfShowGreeting} <p>Hi, welcome to my blog</p> {/block:IfShowGreeting}

Note that the block name should be the meta name truncated. Here, the cheesey message would only be displayed if the content was set to 1. The user can control this visibility by seeing this on their customize page:
